Function Declaration

Declaração tradicional de função que pode ser chamada antes da declaração (hoisting).


// Function Declaration
function saudacao(nome) {
    return `Olá, ${nome}!`;
}

// Chamada da função
console.log(saudacao("Maria")); // "Olá, Maria!"
Function declarations são içadas (hoisted) para o topo do escopo.

Arrow Functions (ES6+)

Sintaxe mais concisa e não vincula seu próprio this.


// Arrow Function básica
const quadrado = (num) => num * num;

// Arrow Function com múltiplos parâmetros
const soma = (a, b) => a + b;

// Arrow Function com corpo de bloco
const fatorial = (n) => {
    if (n <= 1) return 1;
    return n * fatorial(n - 1);
};

console.log(quadrado(5)); // 25
console.log(soma(2, 3));  // 5
console.log(fatorial(4)); // 24

Closures

Funções que "lembram" do ambiente em que foram criadas, mesmo após a execução da função externa.


function criarContador() {
    let contador = 0;
    
    return function() {
        contador++;
        return contador;
    };
}

const meuContador = criarContador();

console.log(meuContador()); // 1
console.log(meuContador()); // 2
console.log(meuContador()); // 3

Boas Práticas

  • Use nomes descritivos para funções
  • Mantenha as funções pequenas e com um único propósito
  • Evite efeitos colaterais quando possível
  • Documente funções complexas com comentários

• Use parâmetros padrão para maior flexibilidade

• Considere usar arrow functions para callbacks

• Teste suas funções com diferentes inputs
